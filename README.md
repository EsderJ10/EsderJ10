## What I'm building

**[Kubeport](https://github.com/EsderJ10/kubeport)** — a Kubernetes control plane built on top of Frappe/ERPNext. Desired state lives in MariaDB through Frappe DocTypes; observed state is queried live from the cluster. The project ships as two repos:

- [**kubeport**](https://github.com/EsderJ10/kubeport) — backend application (Python 3.14+, Frappe v16, Helm 3)
- [**kubeport-deploy**](https://github.com/EsderJ10/tfg) — deployment wrapper (Docker Compose, k3s, devcontainer)
